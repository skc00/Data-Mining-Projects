Data Understanding¶

Import of Dataset¶

The dataset obtained from UC Irvine Machine Learning Repository (https://archive.ics.uci.edu/ml/datasets/Bank+Marketing.), which contains the information related to a direct marketing campaign of a Portuguese bank and marketing staragety via phone calls to get its clients to subscribe for a term deposit.The bank made phone calls to potential clients Often, more than 1 contact to the same client was required to assess whether a client would subscribe for a term deposit or not.The data was collected between May 2008 to November 2010.There are 45,211 observations and 16 independent variables and 1 response variable in the Data Set. There are 7 continuous measure variables and 10 categorical variables. The target response (y) is a binary response indicating whether the client subscribed to a term deposit(numeric value 1) or not(numeric value 0).The objective of our research is to predict if the client will subscribe to term deposit or not(Variable y) and which variables have the highest influence on wheather the client subsribcrition to tem deposit.

Data Understanding¶

There are total 17 attributes in this datasets.The table below describes the variable name, its category, its description, and the type of variable (Continuous or Categorical.)The variables are broken into 4 categories: Client Data, Last Contact Info, Other, and Social and Economic Variables.There are no missing categorical and continuous values in this datset.

def test_accuracy(model, n_splits=3, print_steps=False, params={}):
    accuracies = []
    for i in range(1, n_splits+1):
        X_train, X_test, y_train, y_test = train_test_split(X, y, test_size= 0.2, random_state=None)
        yhat, _ = model(
            X_train=X_train,
            y_train=y_train,
            X_test=X_test,
            **params
        )
        accuracy = float(sum(yhat==y_test)) / len(y_test)
        accuracies.append(accuracy)
        if print_steps:
            matrix = pd.DataFrame(confusion_matrix(y_test, yhat),
                columns=['Predicted 1', 'Predicted 0'],
                index=['Actual 1', 'Actual 0'],
            )
            print('*' * 15 + ' Split %d ' % i + '*' * 15)
            print('Accuracy:', accuracy)
            print(matrix)
            
    return np.mean(accuracies)
    
def find_optimal_accuracy(model, param, param_values, params={}):
    result = {}
    for param_value in tqdm(list(param_values)):
        params_local = params.copy()
        params_local[param] = param_value
        result[param_value] = test_accuracy(model, params=params_local)
    
    result = pd.Series(result).sort_index()
    plt.xlabel(param, fontsize=12)
    plt.ylabel('Accuracy', fontsize=12)
    
    optimal_param = result.argmax()
    optimal_accuracy = result[optimal_param]
    
    if type(param_value) == str:
        result.plot(kind='bar')
    else:
        result.plot()
    plt.show()
    return optimal_param

Advantage of Each Model¶

Advantage in accuracy¶

Logistic regression was performed first time using the built in parameters, and got an accuracy of 0.892. After optimizing for iterations, alpha, epsilon, and L1 and L2 penalties,a final accuracy of 0.894 was obtained which could be an indication of optimizing parameters individually.

SGD requires a number of hyperparameters such as the regularization parameter and the number of iterations.I was suprised when SDG was performed with basic parameters without optimization and got an accuracy of 0.877 (less than that of logistic regression but could be due to lack of optimization). After optimizing, the model was able to achieve an accuracy of 0.887. This is a slight improvement,with still optimizing parameters individually, where interaction between the terms were not considered and also the high risk of overfitting our model.SGD is sensitive to feature scaling.

Advantage in time and efficiency¶

I choose SGDClassifier over other sklearn functions because of time efficiency for our dataset.I tried some other sklearn vector machine functions like SVC setting kernel = 'linear',linear SVC and noticed that accuracy was similar but it was more time consuming. Logistic regression uses the liblinear library and uses a one vs the rest algorithm. This means that the run time is in log linear time hence improve the efficiency.
